# HTML-PRACITCE

  ---
  1. 整体介绍

  这是一个学习进度追踪器，专门帮助想成为"软件测试开发工程师"（SDET）的人制定学习计划。它同时覆盖了三个方面的学习：编程测
  试技能、英语语法、以及技术英语词汇。你可以每天打开这个页面，查看今天该学什么，完成任务后打勾，它会自动记录你的学习天数
  （"连续打卡"），就像一个游戏化的学习日记本。

  ---
  2. 结构拆解

  整个页面可以分成以下几个主要部分：

  - 顶部导航栏 — 显示网站名称和学习天数打卡记录，还有"Google登录"按钮，让你在不同设备上同步数据。
  - 三个学习模块的切换标签 — 你可以点击"SDET"、"English"或"Tech English"来切换不同的学习计划。
  - 任务卡片 — 每个模块最核心的区域，显示当前周的学习任务清单（比如"安装 IntelliJ
  IDEA"），每个任务前面有个小方框可以打勾。
  - 日历和进度环 —
  一个月历格子图，显示你哪几天完成了学习，还有一个圆形的进度环（像苹果手表上的运动环），告诉你整体进度到了百分之多少。
  - 承诺和倒计时按钮 — 你可以写下"我承诺每天晚上7点在书桌前学习"，然后点一个大按钮开始计时学习。
  - 错误记录区 — 一个可以展开的区域，记录你常犯的错误，方便复习。

  3. 关键代码讲解

  我挑了5个对你来说最有代表性的代码片段：

  片段 1：打勾任务（第155-160行）
  .task-checkbox.checked {
    background: var(--green); /* 背景变绿色 */
    border-color: var(--green);
  }
  .task-text.done {
    text-decoration: line-through; /* 文字中间画一条横线 */
    color: var(--text-muted);      /* 颜色变灰 */
  }
  大白话：就像你手机上的待办事项App——点一下任务前面的小方框，它就变成绿色打勾，同时任务文字上会出现一条删除线，颜色变淡
  ，表示"这件事做完了"。这是前端开发里最常见的"切换状态"效果。

  ---
  片段 2：连续打卡记录（第82-85行）
  <div class="streak-badge">
    <svg>🔥图标</svg>
    <span id="streakCount">0</span> day streak
  </div>
  大白话：这就像你玩手机游戏里的"连续签到"——如果你每天都来学习，这个数字就会不断增加。比如显示"5 day
  streak"，意思是"你已经连续学习了5天！"如果中断一天，这个数字就会归零重新开始。这个机制是为了鼓励你每天都来。

  大白话：就像手机App底部的"首页""发现""我的"标签页——你点哪个，哪个就亮起来，对应的内容就显示出来，其他的藏起来。这里你
  点"SDET"就显示编程任务，点"English"就显示英语任务，不会混在一起。

  片段 4：JavaScript 保存你的进度（第477-487行）
  function save(k) {
    localStorage.setItem(p + '_tasks', JSON.stringify(st.tasks));
    localStorage.setItem(p + '_streak', String(st.streak));

  大白话：想象你在一张纸上写日记，然后把纸放进抽屉里收好。localStorage就是浏览器的"抽屉"——每次你打勾一个任务，这段代码就

  4. 它是怎么工作的（故事版）

  想象你是一个要成为软件测试员的学生，叫小明。

  你打开这个页面：

  首先，浏览器的"抽屉"（localStorage）被翻开了——代码检查你昨天的学习记录还在不在、你上次正在学第几周。所有这些信息被快速
  读出来，就像服务员翻看老顾客的档案。

  然后，页面"啪"地一下在你眼前展开。顶部的"连续打卡"徽章显示你已经学习了X天。如果今天是新的一天，它会"叮"地弹出一个小动
  画庆祝你延续了打卡。

  你看到三个标签：SDET、English、Tech English。你点了"SDET"（因为今天是学编程的日子）。页面刷地切换到编程任务面板。

  你看到本周的任务清单：
  - ☐ 安装 IntelliJ IDEA
  - ☐ 写一个 Animal 类和它的子类 Dog、Cat
  - ☐ 创建一个 GitHub 仓库
  - ...等等

  每个任务下面还有一行小字提示具体怎么做。你完成了第一个任务，点击前面的小方框——它变成绿色，文字被划掉。同时，顶部的进度
  条向右移动了一点。这一切都被自动保存到了浏览器的"抽屉"里。

  你回头看日历——今天的格子被标记了一个绿色小勾。圆形进度环也转了一点点。

  如果你想登录Google账号，你的所有进度会同步到云端数据库（Firebase），这样你在公司电脑和家里的电脑上都能看到同样的进度。

  5. 你学到了什么

  这个项目涉及的前端基础知识（每个用一句话解释）：

  - HTML（超文本标记语言） — 像房子的骨架，用来定义页面上有什么东西：标题、按钮、列表、输入框等。
  - CSS（层叠样式表） — 像房子的装修和油漆，控制颜色、大小、位置、动画等所有视觉外观。
  - JavaScript — 像房子的电力和智能系统，让页面能"动起来"：点击按钮有反应、保存数据、计算进度等。
  - localStorage（本地存储） — 浏览器自带的小笔记本，可以在你电脑上保存数据，关掉浏览器也不会丢。
  - Firebase（谷歌的后端服务） — 一个远程服务器服务，让你可以登录账号，在不同设备之间同步学习数据。
  - 事件监听（Event Listener） — 就像给按钮装了一个"耳朵"，当用户点击时触发相应的动作。
  - CSS动画（Animation） — 让元素在一段时间内平滑变化（比如进度条从 0% 移到 50%），而不是"啪"地一下跳过去。
  - 响应式设计 — 让页面在不同大小的屏幕上都能正常显示，不管是手机、平板还是电脑。
